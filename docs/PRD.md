# 產品需求文件 (PRD) - 活動報名系統

## 1. 專案概述
- **背景與動機**：計畫經理人需要一個有效率的工具來建立活動並掌握報名狀況，特別是能夠準確統計報名人數及確認報名者名單，以利後續作業。
- **目標用戶**：計畫經理人（活動建立與管理者）。
- **核心價值主張**：提供一個直覺且簡單的管理系統，讓計畫經理人能快速建立、說明與整理活動內容，並自動化統計報名人數與名單，大幅降低行政負擔。

## 2. 功能需求

### 主要功能清單
1. **建立活動**：系統提供介面供建立新的活動。
2. **說明活動內容**：允許輸入與展示詳細的活動資訊（如時間、地點、議程等）。
3. **整理內容**：可以修改、更新或下架已建立的活動內容。
4. **統計人數**：系統能自動計算並顯示各個活動目前的報名總數。
5. **確認人員名單**：提供檢視詳細報名人員資料的列表功能。

### 使用者故事 (User Stories)
- **作為** 計畫經理人，**我希望** 能在系統上建立活動，**以便** 開放給目標對象報名。
- **作為** 計畫經理人，**我希望** 能詳細說明活動內容，**以便** 讓參與者清楚了解並吸引他們報名。
- **作為** 計畫經理人，**我希望** 能整理及編輯現有活動內容，**以便** 隨時更新並維持資訊正確性。
- **作為** 計畫經理人，**我希望** 系統能自動統計活動報名人數，**以便** 快速掌握狀況並評估是否達到預期目標。
- **作為** 計畫經理人，**我希望** 能確認報名該活動的人員詳細名單，**以便** 進行活動前的聯絡、審核或當日的報到準備。

## 3. 非功能需求
- **技術限制**：
  - 後端框架：Flask
  - 前端模板：Jinja2 + 基本 HTML/CSS
  - 資料庫：SQLite
- **效能與安全考量**：
  - 確保表單提交時的資料驗證，避免空值或不合法的資料寫入資料庫。
  - 防範常見的網路攻擊（如 SQL Injection 及 XSS），使用 SQLAlchemy 等 ORM 搭配 Jinja2 預設的跳脫（escaping）機制。
  - 確保報名統計的即時性與一致性。

## 4. MVP 範圍

### Must Have (必須擁有)
- 計畫經理人建立、編輯與刪除活動的功能。
- 可填寫詳細活動說明的功能。
- 提供給參與者報名的表單頁面。
- 管理後台能即時統計並顯示報名總人數。
- 管理後台能列表展示並確認報名人員名單。

### Should Have (應該擁有)
- 設定活動人數上限，額滿時自動停止接受報名。
- 登入機制，保護管理後台僅供授權的計畫經理人使用。

### Nice to Have (最好擁有)
- 匯出人員名單至 Excel 或 CSV。
- 活動類別標籤或是搜尋過濾功能。
